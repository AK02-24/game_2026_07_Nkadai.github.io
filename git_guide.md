# 初めてのGitHubコミット＆プッシュガイド

このガイドでは、初めてGitやGitHubを使う方に向けて、ファイルの変更をGitHubに保存（コミット＆プッシュ）する方法を分かりやすく解説します。

---

## 🚀 最も簡単な方法：`git_helper.bat` を使う

同じフォルダに作成した [git_helper.bat](file:///d:/AI_お試し/ネットワーク課題/ネットワーク課題/git_helper.bat) をダブルクリックするだけで、画面の指示に従ってコミットとプッシュを行うことができます。

### 実行手順：
1. [git_helper.bat](file:///d:/AI_お試し/ネットワーク課題/ネットワーク課題/git_helper.bat) をダブルクリックして実行します。
2. **Gitの設定確認**: ユーザー名とメールアドレスが表示されます。変更したい場合は `y` を押し、新しい情報を入力します。
3. **ステージングの確認**: 変更したファイルをコミット対象に追加するか尋ねられるので、`y` を入力します。
4. **コミットメッセージの入力**: 「何を変更したか」を短い言葉で入力します（例: `最初のコミット` や `index.htmlの修正` など）。
5. **プッシュ**: 自動的にGitHubへアップロード（プッシュ）が実行されます。

## 💻 コマンドプロンプトやPowerShellで手動で行う方法

もしコマンドを手動で打ちたい場合は、以下の3つのステップを行います。

### 1. 変更したファイルをコミット対象にする（ステージング）
```bash
git add .
```
※ `.` は「すべての変更されたファイル」を指します。特定のファイルだけを指定したい場合は `git add index.html` のように指定します。

### 2. ローカルに保存する（コミット）
```bash
git commit -m "ここにコミットメッセージ"
```
※ メッセージは、後から履歴を見たときに何を変更したかが分かるように書きます。

### 3. GitHubに送信する（プッシュ）
```bash
git push origin main
```
※ 初回プッシュの際、GitHubへのサインインを求められた場合は、ブラウザでログインして認証を行ってください。

## 🛠️ よくあるエラーと対処法

### 1. `Permission denied (publickey)` エラーが出る
GitHubへのアクセスにSSH接続（`git@github.com:...`）を使用しているが、PCにSSHキーが設定されていないか、GitHubに登録されていない場合に発生します。

**対処法：**
1. すでにSSHキーがあるか確認、または新規作成します。
2. GitHubの「Settings」→「SSH and GPG keys」に、あなたのPCの公開鍵（`id_rsa.pub` などの中身）を登録します。

※ または、接続方法をHTTPS（`https://github.com/...`）に変更することでも解決できます。

### 2. `Updates were rejected because the remote contains work...` と出る
GitHub（リモート）側に、あなたのPC（ローカル）にはない新しいコミットが存在する場合に発生します。

**対処法：**
プッシュする前に、まずGitHub上の最新の状態を取り込みます（プル）。
```bash
git pull origin main
```
プルを実行した後、再度プッシュを行います。

### 3. ユーザー名とメールアドレスの警告が出る
コミットした際に、誰がコミットしたかを記録するための設定がされていない場合です。

**対処法：**
ターミナルで以下のコマンドを実行して設定してください（`git_helper.bat` でも自動で確認・設定できます）。
```bash
git config --global user.name "あなたのユーザー名"
git config --global user.email "あなたのメールアドレス"
```
